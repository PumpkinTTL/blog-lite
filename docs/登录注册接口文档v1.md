# 登录注册接口文档

> 基础路径：`/v1/user`  
> 认证方式：JWT Bearer Token  
> 所有接口均为 POST  
> 响应码规范：200=成功，4xx=客户端错误，5xx=服务端错误

---

## 接口总览

| 接口 | 认证 | 说明 |
|------|------|------|
| `/v1/user/login` | 无 | 密码登录 / 验证码登录 |
| `/v1/user/register` | 无 | 用户名+密码注册 |
| `/v1/user/sendEmailCode` | 无 | 发送邮箱验证码 |
| `/v1/user/logout` | ✅ | 退出登录 |
| `/v1/user/refreshToken` | ✅ | Token 续签 |
| `/v1/user/changePassword` | ✅ | 修改密码 |
| `/v1/user/requestPasswordReset` | 无 | 请求密码重置邮件 |
| `/v1/user/verifyResetToken` | 无 | 校验重置令牌 |
| `/v1/user/resetPassword` | 无 | 重置密码 |

---

## 1. 密码登录

`POST /v1/user/login`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | ✅ | 用户ID / 邮箱 / 手机号 |
| password | string | ✅ | 密码 |
| action | string | ✅ | 固定 `pwd` |

**请求示例：**

```json
{
  "account": "admin",
  "password": "123456",
  "action": "pwd"
}
```

**成功响应：**

```json
{
  "code": 200,
  "msg": "密码登录成功",
  "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "expireTime": 1700259200,
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "管理员",
    "avatar": "",
    "status": true,
    "roles": ["admin"],
    "premium": null
  }
}
```

**错误码：**

| code | 说明 |
|------|------|
| 400 | 参数错误 |
| 401 | 账号或密码错误 |
| 403 | 用户被封禁 |

## 2. 邮箱验证码登录（自动注册）

`POST /v1/user/login`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | ✅ | 邮箱地址 |
| password | string | ✅ | 邮箱验证码（6位数字） |
| action | string | ✅ | 固定 `code` |
| invite_code | string | 新用户必填 | 邀请码（老用户无需传） |

**请求示例：**

```json
{
  "account": "user@example.com",
  "password": "123456",
  "action": "code",
  "invite_code": "ABC123"
}
```

**流程说明：**

1. 先调用 `sendEmailCode` 发送验证码到邮箱
2. 用验证码作为 password 登录，action 传 `code`
3. **老用户**：直接登录成功，返回 token
4. **新用户**：自动注册并登录，**必须提供 invite_code**

**成功响应：**

```json
{
  "code": 200,
  "msg": "登录成功",
  "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "expireTime": 1700259200,
  "data": {
    "id": 2,
    "username": "user_1700000000",
    "nickname": "user",
    "avatar": "",
    "status": true,
    "roles": ["普通用户"],
    "premium": null
  }
}
```

**错误码：**

| code | 说明 |
|------|------|
| 400 | 验证码错误或已过期 / 注册需要邀请码 |
| 403 | 用户已被禁用 |
| 500 | 自动注册失败 |

---

## 3. 发送邮箱验证码

`POST /v1/user/sendEmailCode`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | ✅ | 邮箱地址 |

**请求示例：**

```json
{
  "email": "user@example.com"
}
```

**成功响应：**

```json
{
  "code": 200,
  "msg": "验证码已发送"
}
```

**说明：**
- 验证码为 6 位数字
- 有效期 5 分钟
- 同一邮箱发送后旧验证码立即失效

## 4. 用户名注册

`POST /v1/user/register`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | ✅ | 用户名（3-255字符，唯一） |
| password | string | ✅ | 密码（≥6字符） |
| email | string | ✅ | 邮箱（唯一） |
| nickname | string | ❌ | 昵称，默认=username |
| gender | int | ❌ | 0=未知 1=男 2=女，默认=2 |

**请求示例：**

```json
{
  "username": "testuser",
  "password": "123456",
  "email": "test@example.com",
  "nickname": "测试用户"
}
```

**成功响应：**

```json
{
  "code": 200,
  "msg": "注册成功",
  "data": {
    "id": 3,
    "username": "testuser",
    "email": "test@example.com",
    "roles": [{ "id": 18, "name": "普通用户" }]
  }
}
```

**错误码：**

| code | 说明 |
|------|------|
| 400 | 参数错误 |
| 409 | 用户名已存在 / 邮箱已被使用 |
| 500 | 注册失败 |

```
Authorization: Bearer {token}
```

**成功响应：**

```json
{
  "code": 200,
  "msg": "注销成功"
```

## 6. Token 续签

`POST /v1/user/refreshToken`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | string | ✅ | 当前有效的 Token |

**成功响应：**

```json
{
  "code": 200,
  "msg": "续签成功",
  "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "expireTime": 1700259200
}
```

**错误码：**

| code | 说明 |
|------|------|
| 400 | 参数错误 |
| 401 | Token验证失败 / 已失效，请重新登录 |
| 403 | 用户已被禁用 |
| 404 | 用户不存在 |

## 7. 修改密码

`POST /v1/user/changePassword`

**请求头：**

```
Authorization: Bearer {token}
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| old_password | string | ✅ | 旧密码 |
| new_password | string | ✅ | 新密码（≥6字符） |

**成功响应：**

```json
{
  "code": 200,
  "msg": "密码修改成功，请重新登录"
}
```

> 修改成功后 Token 自动失效，需重新登录。

## 8. 密码重置

三步流程，无需登录。

### Step 1：请求重置

`POST /v1/user/requestPasswordReset`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | ✅ | 注册邮箱 |

成功后发送重置链接到邮箱，10分钟有效。

### Step 2：校验令牌

`POST /v1/user/verifyResetToken`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | string | ✅ | 重置令牌 |

返回邮箱脱敏信息，确认令牌有效。

### Step 3：重置密码

`POST /v1/user/resetPassword`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | string | ✅ | 重置令牌 |
| new_password | string | ✅ | 新密码（≥6字符） |

成功后强制清除旧 Token，需重新登录。

---

## Token 说明

- **算法**：HS256
- **有效期**：3 天（259200 秒）
- **存储**：JWT + Redis 双重校验
- **携带方式**（优先级从高到低）：
  1. `Authorization: Bearer {token}` 请求头
  2. `Authorization` 请求头（无 Bearer 前缀）
  3. Cookie `token`

**Token 过期处理：**
- `code: 401` → Token 无效/已过期/被顶替，需重新登录
- Token 即将过期时调用 `refreshToken` 续签

---

## 响应码规范

| code | 含义 | 场景 |
|------|------|------|
| 200 | 成功 | 所有成功操作 |
| 400 | 参数错误 / 请求无效 | 缺少必填参数、验证码错误、格式不对 |
| 401 | 未授权 | Token无效、账号密码错误、Token过期 |
| 403 | 禁止访问 | 用户被封禁、权限不足、旧密码错误 |
| 404 | 资源不存在 | 用户不存在、等级记录不存在 |
| 409 | 资源冲突 | 用户名已存在、邮箱已被使用 |
| 500 | 服务器内部错误 | 代码异常、数据库操作失败 |
