# 客户端接口文档 (V2)

> 基础路径：`/v2/user`  
> V2 为客户端专用接口，数据经过安全过滤，不含管理敏感字段  
> 认证方式：JWT Bearer Token  
> 所有接口均为 POST  
> 响应码规范：200=成功，4xx=客户端错误，5xx=服务端错误


## 接口总览

### 公开接口（无需登录）

| 接口 | 说明 |
|------|------|
| `/v2/user/login` | 密码登录 / 验证码登录 |
| `/v2/user/register` | 用户名+密码注册 |
| `/v2/user/sendEmailCode` | 发送邮箱验证码 |
| `/v2/user/checkUsername` | 检查用户名是否可用 |
| `/v2/user/checkEmail` | 检查邮箱是否可用 |
| `/v2/user/requestPasswordReset` | 请求密码重置邮件 |
| `/v2/user/verifyResetToken` | 校验重置令牌 |
| `/v2/user/resetPassword` | 重置密码 |
| `/v2/user/refreshToken` | Token 续签 |

### 需要登录的接口

| 接口 | 说明 |
|------|------|
| `/v2/user/profile` | 获取当前用户信息（安全过滤版） |
| `/v2/user/update` | 更新个人资料（仅安全字段） |
| `/v2/user/changePassword` | 修改密码 |
| `/v2/user/membership` | 获取会员状态 |
| `/v2/user/delete` | 注销账号（软删除） |
| `/v2/user/logout` | 退出登录 |

---

## Token 传递方式

支持 Header 和 Cookie 两种形式，**完全兼容**，按优先级依次尝试：

| 优先级 | 方式 | 示例 |
|--------|------|------|
| 1 | `Authorization` 请求头（Bearer 格式） | `Authorization: Bearer eyJ0eXAi...` |
| 2 | `Authorization` 请求头（直接传 Token） | `Authorization: eyJ0eXAi...` |
| 3 | `Bearer` 自定义请求头 | `Bearer: eyJ0eXAi...` |
| 4 | Cookie `Authorization` | `Cookie: Authorization=eyJ0eXAi...` |
| 5 | Cookie `token` | `Cookie: token=eyJ0eXAi...` |

**客户端建议**：Web 端用 `Authorization: Bearer {token}`，移动端/H5 不方便设 Header 的走 Cookie 也可以。

- 算法：HS256
- 有效期：3 天（259200 秒）
- 存储方式：JWT + Redis 双重校验

## 1. 密码登录

`POST /v2/user/login`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | ✅ | 用户ID / 邮箱 / 手机号 |
| password | string | ✅ | 密码 |
| action | string | ✅ | 固定 `pwd` |

**请求示例：**

```json
{
  "account": "admin",
  "password": "123456",
  "action": "pwd"
}
```

**成功响应：**

```json
{
  "code": 200,
  "msg": "密码登录成功",
  "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "expireTime": 1700259200,
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "管理员",
    "email": "admin@example.com",
    "avatar": "",
    "gender": 1,
    "signature": "",
    "phone": "138*****888",
    "status": true,
    "last_login": "2025-04-11 16:00:00",
    "register_source": "web",
    "roles": [{ "name": "管理员", "iden": "admin", "description": "系统管理员" }],
    "premium": null
  }
}
```

**错误码：**

| code | 说明 |
|------|------|
| 400 | 参数错误 |
| 401 | 账号或密码错误 |
| 403 | 用户被封禁 |

## 2. 邮箱验证码登录（自动注册）

`POST /v2/user/login`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | ✅ | 邮箱地址 |
| password | string | ✅ | 邮箱验证码（6位数字） |
| action | string | ✅ | 固定 `code` |
| invite_code | string | 新用户必填 | 邀请码（老用户无需传） |

**流程：**
1. 先调 `sendEmailCode` 发验证码
2. 用验证码作为 password 登录，action 传 `code`
3. 老用户直接登录，新用户自动注册（必须提供 invite_code）

**错误码：**

| code | 说明 |
|------|------|
| 400 | 验证码错误或已过期 / 注册需要邀请码 |
| 403 | 用户已被禁用 |
| 500 | 自动注册失败 |

---

## 3. 发送邮箱验证码

`POST /v2/user/sendEmailCode`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | ✅ | 邮箱地址 |

- 验证码 6 位数字，有效期 5 分钟，发送后旧验证码立即失效

## 4. 用户名注册

`POST /v2/user/register`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | ✅ | 用户名（3-255字符，唯一） |
| password | string | ✅ | 密码（≥6字符） |
| email | string | ✅ | 邮箱（唯一） |
| nickname | string | ❌ | 昵称，默认=username |
| gender | int | ❌ | 0=未知 1=男 2=女，默认=2 |

**错误码：**

| code | 说明 |
|------|------|
| 400 | 参数错误 |
| 409 | 用户名已存在 / 邮箱已被使用 |
| 500 | 注册失败 |

## 7. 获取当前用户信息

`POST /v2/user/profile`

**需要登录** ✅ — 自动从 JWT 获取当前用户，无需传 ID

**成功响应：**

```json
{
  "code": 200,
  "msg": "获取用户信息成功",
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "管理员",
    "email": "admin@example.com",
    "avatar": "",
    "gender": 1,
    "signature": "",
    "phone": "138*****888",
    "status": true,
    "last_login": "2025-04-11 16:00:00",
    "register_source": "web",
    "roles": [
      { "name": "管理员", "iden": "admin", "description": "系统管理员" }
    ],
    "level_records": [
      { "target_type": "user", "level": 5, "experience": 1200 }
    ],
    "premium": {
      "id": 1,
      "user_id": 1,
      "expiration_time": "2099-01-01 00:00:00",
      "is_active": true,
      "remark": "永久会员"
    }
  }
}
```

**与 V1 selectUserInfoById 的区别：**
- 不需要传 `targetUid`，只能查自己
- 不返回 `last_login_ip`、`create_time`、`update_time`、`invite_code`、`register_device` 等管理字段
- `premium` 额外返回 `is_active` 字段（客户端可直接判断会员是否生效）

## 8. 更新个人资料

`POST /v2/user/update`

**需要登录** ✅ — 只允许更新安全字段

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| nickname | string | ❌ | 昵称（最长255字符） |
| gender | int | ❌ | 0=未知 1=男 2=女 |
| signature | string | ❌ | 个性签名（最长500字符） |
| avatar | string | ❌ | 头像URL |

> 不允许修改 username、email、phone、password、status、角色等字段

**成功响应：** 返回更新后的完整用户数据（同 profile 格式）

## 9. 修改密码

`POST /v2/user/changePassword`

**需要登录** ✅

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| old_password | string | ✅ | 旧密码 |
| new_password | string | ✅ | 新密码（≥6字符） |

> 修改成功后 Token 自动失效，需重新登录。

## 11. 注销账号

`POST /v2/user/delete`

**需要登录** ✅ — 软删除，30天内可恢复

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| password | string | ✅ | 当前密码（确认身份） |
| confirmation | string | ✅ | 固定文本 `我确认删除账号` |

> 注销后 Token 立即失效，所有关联数据软删除

---

## 12. 退出登录

`POST /v2/user/logout`

**需要登录** ✅

清除 Redis Token，使当前 Token 立即失效。

---

## 13. 密码重置

三步流程，无需登录。

### Step 1：请求重置

`POST /v2/user/requestPasswordReset`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | ✅ | 注册邮箱 |

### Step 2：校验令牌

`POST /v2/user/verifyResetToken`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | string | ✅ | 重置令牌 |

### Step 3：重置密码

`POST /v2/user/resetPassword`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | string | ✅ | 重置令牌 |
| new_password | string | ✅ | 新密码（≥6字符） |

---

## 响应码规范

| code | 含义 | 场景 |
|------|------|------|
| 200 | 成功 | 所有成功操作 |
| 400 | 参数错误 / 请求无效 | 缺少必填参数、验证码错误、格式不对 |
| 401 | 未授权 | Token无效/过期、账号密码错误 |
| 403 | 禁止访问 | 用户被封禁、权限不足 |
| 404 | 资源不存在 | 用户不存在 |
| 409 | 资源冲突 | 用户名已存在、邮箱已被使用 |
| 500 | 服务器内部错误 | 代码异常、数据库操作失败 |
